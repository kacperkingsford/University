Zadanie 1

Autor: Jacek Kowalski

Czy poniższe wyrażenia zawsze obliczą się do prawdy dla dwóch dowolnych wartości zmiennych «x» i «y» typu «int32_t»? Jeśli nie to podaj wartości, które prowadzą do obliczenia fałszu.
• (x > 0) || (x - 1 < 0)
• (x & 7) != 7 || (x << 29 < 0)
• (x * x) >= 0
• x < 0 || -x <= 0
• x > 0 || -x >= 0
• (x | -x) >> 31 == -1
• x + y == (uint32_t)y + (uint32_t)x
• x * ~y + (uint32_t)y * (uint32_t)x == -x

  • (x > 0) || (x - 1 < 0)
    • x = INT_MIN
  • (x & 7) != 7 || (x << 29 < 0)
    • Zawsze spełnione: 0…0111 => 1110…0
  • (x * x) >= 0
    • x = USHRT_MAX
  • x < 0 || -x <= 0
    • Zawsze spełnione
  • x > 0 || -x >= 0
    • x = INT_MIN
  • (x | -x) >> 31 == -1
    • x = 0
  • x + y == (uint32_t)y + (uint32_t)x
    • Zawsze spełnione co do reprezentacji bitowej
  • x * ~y + (uint32_t)y * (uint32_t)x == -x
    • Zawsze spełnione

Do ostatniego punktu:
x * (~y + y) == x * -1

Napisz ciąg instrukcji, który bez użycia dodatkowych zmiennych, zamieni miejscami zawartość zmiennych «x» i «y».
Wskazówka: Spróbuj rozwiązać zadanie samodzielnie, a następnie przeczytaj §2.19 książki „Uczta programistów”.




x = x ^ y;
y = y ^ x;
x = x ^ y;

Czemu to działa?

  • x = x ^ y
  • y = y ^ (x ^ y) = (xor jest rozłączny i przemienny) = (y ^ y) ^ x = 0 ^ x = x
  • x = x ^ y = (x ^ y) ^ x = y ^ (x ^ x) = y ^ 0 = y

Zadanie 4

Autor: Mateusz Leonowicz

Zmienne « x » i « y » przechowują liczby typu « uint32_t » składające się z czterech bajtów, tj. x=∑3i=0xi⋅28ix=∑i=03xi⋅28i oraz y=∑3i=0yi⋅28iy=∑i=03yi⋅28i. Jak szybko obliczyć z=∑3i=0zi⋅28iz=∑i=03zi⋅28i gdzie zi=xi⊕yizi=xi⊕yi , gdy:
• ⊕ jest operacją dodawania,
• ⊕ jest operacją odejmowania.
Obliczając wynik należy zapobiec wystąpieniu przeniesienia (ang. carry) lub pożyczki (ang. borrow) propagującego się do bardziej znaczącego bajtu.
Wskazówka: Spróbuj rozwiązać zadanie samodzielnie, a następnie przeczytaj §2.17 książki „Uczta programistów”.

Dodawanie:

Dodajemy wszystkie bity, poza tymi najbardziej znaczącymi, aby się nie rozpropagowały

r = (x & 0x7F7F7F7F) + (y & 0x7F7F7F7F)

Teraz musimy dodać bit znaczący, ale tylko pod warunkiem, że nie został już on tam włączony i że nie jest on włączony jednocześnie w x i y

r = r ^ ((x ^ y) & 0x80808080)

Odejmowanie:

Rozważmy przypadki, pokażę że jest to poprawne rozwiązanie




r = (x | 0x80808080) - (y & 0x7F7F7F7F)
r = ((~(x ^ y) & 0x80808080) ^ r)

Jeśli bit znaczący w x = 1, a w y = 1
To najpierw odejmujemy od siebie zwykczajnie liczby, oprócz najstarszego bitu, czyli 2^7
Czyli robimy zwykłe odejmowanie na początku, ale musimy jeszcze odjąć brakujące 2^7. Czyli flipujemy najstarszy bit.

Jeśli bit znaczący w x = 1, a w y = 0
Jest to zwyczajne odejmowanie.

Jeśli bit znaczący w x = 0, a w y = 0
Dodajemy do x 2^7, a na końcu je odejmujemy, więc wynik się zgadza.

Jeśli bit znaczący w x = 0, a w y = 1
Dodajemy do x 2^7, a z y zabieramy 2^7. Na koniec musimy w takim razie zabrać 2*2^7 = 2^8, ale ponieważ jest to za duża liczba, to nic to nie zmieni.

Zadanie 5

Autor: Paweł Zmarzły

Uzupełnij ciało funkcji zadeklarowanej następująco:

/* Oblicz x * 3 / 4 zaokrąglając w dół. */
int32_t threefourths(int32_t x);

Nie można dopuścić do wystąpienia nadmiaru i niedomiaru!























int32_t threefourths(int32_t x) {
  return (x >> 1) + (x >> 2) + (x & (x >> 1) & 1);
}

int main(void) {
  int testcase = -INT32_MAX;
  printf("Wynik przy liczeniu na longach: %lld\n",
         (long)(testcase)*3 / 4);
  printf("Wynik double: %lf\n", (double)(testcase)*3. / 4.);
  printf("Wynik double (floor): %lld\n",
         (long)floor((double)(testcase)*3. / 4.));
  printf("Nasz wynik: %d\n", threefourths(testcase));

  for (long i = INT32_MIN; i <= INT32_MAX; i++) {
    if (threefourths(i) != floor(i * 3. / 4.)) {
      printf("%ld\n", i);
      assert(false);
    }
  }

  return 0;
}

Zadanie 7

Autor: Marek Padlewski

Podaj fragment kodu, który oblicza funkcję:
abs(x)={xdla x≥0−xdla x<0abs(x)={xdla x≥0−xdla x<0

Skorzystaj z następującej własności: jeśli « b » jest wartością logiczną, to wyrażenie b ? x : y można przetłumaczyć do b * x + !b * y.
Wskazówka: Spróbuj rozwiązać zadanie samodzielnie, a następnie przeczytaj §2.4 książki „Uczta programistów”.

Własność: b ? x : y możemy przetłumaczyć do b * x + !b * y

Rozwiązanie używające wskazówki (nielegalne):










// b = 0 jeśli x jest >= 0
// b = 1 jeśli x jest <  0

int abs (int x) {
    y = -x;
    b = (x >> 31) & 1
 return !b * x + b * y;
}

Rozwiązanie z książki (nielegalne):




int abs (int x) {
    return ((x >> 30) | 1) * x;
}

Przesuwając xx o 30 bitów i ORując z 11 otrzymamy 1111⋯11111111⋯1111 dla liczby ujemnej lub 0000⋯00010000⋯0001 dla liczby dodatniej.
Czyli ostatecznie robimy (−1)×x(−1)×x jeśli x<0x<0 lub 1×x1×x jeśli x≥0x≥0.

Rozwiązanie nie używające wskazówki:







int abs (int x) {
    int mask = x >> 31;
    x ^= mask;
    x -= mask;
    return x;
}

Przesuwając xx o 31 bitów otrzymujemy liczbę binarną 1111⋯11111111⋯1111 jeśli x<0x<0 lub liczbę 0000⋯00000000⋯0000 jeśli x≥0x≥0.
XORowanie z maską 1111⋯11111111⋯1111 zadziała jak ~xx.
XORowanie z maską 0000⋯00000000⋯0000 nie zmieni wartości xx.

Korzystając ze wzoru -x = ~x + 1 widzimy, że aby otrzymać −x−x musimy do wyniku dodać 11. Tak więc odejmując maskę 1111⋯11111111⋯1111 której wartość to −1−1 otrzymamy to co chcemy.

int32_t sign(int32_t x)
{
    return (x >> 31) - (-x >> 31);
}

Dla x > 0:

    x >> 31 = 0
    -x >> 31 = -1 
    0-(-1) = 1

Dla x = 0:

x >> 31 = 0
-x >> 31 = 0 
0-(-0) = 0

Dla x < 0:

x >> 31 = -1
-x >> 31 = 0
-1 -(0) = -1

Zadanie 9

Autor: Karol Guzikowski

Uzupełnij ciało funkcji zadeklarowanej poniżej.

/* Kiedy x zawiera nieparzystą liczbę jedynek zwróć 1, w p.p. 0 */ 
int32_t odd_ones(uint32_t x);

Wskazówka: Spróbuj rozwiązać zadanie samodzielnie, a następnie przeczytaj §5.2 książki „Uczta programistów”.

https://en.wikipedia.org/wiki/Hamming_weight
















bool odd_ones(uint32_t x){
    uint32_t m1  = 0x55555555; // binary: 0101...
    uint32_t m2  = 0x33333333; // binary: 00110011..
    uint32_t m4  = 0x0f0f0f0f; // binary:  4 zeros,  4 ones...
    uint32_t m8  = 0x00ff00ff; // binary:  8 zeros,  8 ones...
    uint32_t m16 = 0x0000ffff; // binary: 16 zeros, 16 ones
    
    x = (x & m1 ) + ((x >>  1) & m1 ); // put count of each  2 bits into those  2 bits 
    x = (x & m2 ) + ((x >>  2) & m2 ); // put count of each  4 bits into those  4 bits 
    x = (x & m4 ) + ((x >>  4) & m4 ); // put count of each  8 bits into those  8 bits 
    x = (x & m8 ) + ((x >>  8) & m8 ); // put count of each 16 bits into those 16 bits 
    x = (x & m16) + ((x >> 16) & m16); // put count of each 32 bits into those 32 bits
    
    return x & 1;
}

Inaczej:











bool odd_ones(uint32_t x)
{
    x ^= x >> 1;
    x ^= x >> 2;
    x ^= x >> 4;
    x ^= x >> 8;
    x ^= x >> 16;
    
    return x & 1;
}

Powyższe funkcje zwracają true jeśli ta liczba jedynek liczba jest nieparzysta
