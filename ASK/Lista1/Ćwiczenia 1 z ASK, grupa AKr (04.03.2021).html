Autor: Damian Kowalik

Zmienne ii , kk spełniają warunek 0≤i,k≤310≤i,k≤31. Napisz fragment kodu, który skopiuje ii-ty bit zmiennej xx na pozycję kk-tą.







uint32_t solve(uint32_t x, int i, int k)
{
    bool bit = (x >> i) & 1u; // kopiujemy i-ty bit
    x &= ~(1u << k);           // zerujemy k-ty bit
    return x | (bit << k);    // na k-te miejsce wpisujemy i-ty bit
}

Zadanie 2

Autor: Krystian Sańczyk

Napisz fragment kodu, który wyznaczy liczbę zapalonych bitów w zmiennej xx.

UWAGA! Oczekiwana złożoność to O(log(n))O(log(n)), gdzie nn to liczba bitów w słowie. Posłuż się strategią „dziel i zwyciężaj”.















const uint32_t m1  = 0x55555555; // 010101010101...
const uint32_t m2  = 0x33333333; // 001100110011...
const uint32_t m4  = 0x0f0f0f0f; // 000011110000...
const uint32_t m8  = 0x00ff00ff; // 0000000011111111...
const uint32_t m16 = 0x0000ffff; // 16 zer, 16 jedynek ...

int count_bits(uint32_t x) {
    x = (x & m1 ) + ((x >>  1) & m1 );
    x = (x & m2 ) + ((x >>  2) & m2 );
    x = (x & m4 ) + ((x >>  4) & m4 ); 
    x = (x & m8 ) + ((x >>  8) & m8 ); 
    x = (x & m16) + ((x >> 16) & m16);
    return x;
}

Zadanie 4

Autor: Kacper Gawdziński

Rozważamy słowa kluczowe ze standardu C11 (a nie C++). Jakie jest działanie «volatile» w stosunku do zmiennych? Kiedy programiści muszą go użyć, by program zachowywał się poprawnie? Jaki jest skutek użycia « static » w stosunku do zmiennych globalnych, zmiennych lokalnych i procedur? Kiedy należy go używać? Jaką rolę pełni «restrict» odnośnie typów wskaźnikowych?

  • volatile - poza wyłączeniem optymalizacji jest to sposób na powiedzenie kompilatorowi, że dana zmienna może zmieniać się niezależnie od naszego programu.
    Przykład:




int x = keyboard.input
while(x ! = 0) {
    ...
}

  • static - odnośnie zmiennyych globalnych oraz funkcji ogranicza ich scope do aktualnego pliku. Odnośnie zmiennych ogranicza ich zasięg do bloku, jednak nie zwalnia ich pamięci po jego wykonaniu, tylko po zakończeniu programu. Odnośnie klas, tworzy jedną instancję zmiennej która jest dzielona między inne jej obiekty.

  • restrict - mówi o tym, że tylko ten jedyny pointer jest w stanie dotrzeć do obiektu na który wskazuje, tzn nie ma jego kopii. Eliminuje to sytuację aliasowania pointerów, więc kompilator nie musi ich przeładowywać do każdego wystąpienia







void sum_twice(int* a, int* restrict b) {
  *a += *b;
  // Dzięki restrict na b, kompilator zakłada, że wartość
  // *b nie uległa zmianie (nie wywołano sum_twice(a, a)).
  *a += *b;
}

Zadanie 7

Autor: Karol Guzikowski

Przetłumacz, krok po kroku, poniższą procedurę napisaną w języku C na kod trójkowy:













void insertion_sort(int arr[], int length) {
    int j, temp;
    for (int i = 0; i < length; i++) {
        j = i;
        while (j > 0 && arr[j] < arr[j-1]) {
            temp = arr[j];
            arr[j] = arr[j-1];
            arr[j-1] = temp;
            j--;
        }
    }
}

Następnie oznacz bloki podstawowe i narysuj graf przepływu sterowania (ang. control flow graph).

Niech:



x := arr[i] == p := arr + i*4; x := *p
arr[i] := x == p := arr + i*4; *p := x

Czyli:

















            i := 0                        # B1
FOR_START   if i >= length goto F_END     # B2
            j := 1                        # B3
WHILE_START if j <= 0 goto WHILE_END      # B4
            x1 := arr[j]                  # B5
            p := j-1
            x2 := arr[p]
            if x1 < x2 goto WHILE_END
            temp := x1                    # B6
            arr[j] := x2
            arr[p] := temp

            goto WHILE_START
WHILE_END   i=i+1                         # B7
            goto FOR_START
F_END                                     # B8

Blok podstawowy to taki ciąg instrukcji, który ma tylko jedno wejście i tylko jedno wyjście, przy czym wejście do bloku znajduje się na jego początku, a wyjście na końcu.

Zadanie 8

Autor:

Być może jest to zaskakujące, ale poniższy kod jest poprawny i w dodatku czasami korzysta się z tej niskopoziomowej techniki optymalizacji. Co robi procedura «secret»?















void secret(uint8_t *to, uint8_t *from, size_t count) {
    size_t n = (count + 7) / 8;
    switch (count % 8) {
        case 0: do {*to++ = *from++;
        case 7:     *to++ = *from++;
        case 6:     *to++ = *from++;
        case 5:     *to++ = *from++;
        case 4:     *to++ = *from++;
        case 3:     *to++ = *from++;
        case 2:     *to++ = *from++;
        case 1:     *to++ = *from++;
                } while (--n > 0);
    }
}

Kompilator GCC dopuszcza by instrukcja «goto» przyjmowała wyrażenie obliczające adres skoku. Dodatkowo umożliwia definiowanie tablic etykiet 1 . Przetłumacz powyższą procedurę tak, by korzystała wyłącznie z instrukcji «goto».

Procedura secret kopiuje count elemnetów z from do to odpowiednio na pozycje o tych samych indeksach.

Instrukcja switch przenosi nas do odpowiedniego miejsca w kodzie, więc również w środek pętli while (jak użycie goto case_0; na przykład).

Po wykonanych instrukcjach z danego case nie mamy użytego słowa kluczowego break, więc instrukcje z pozostałych case’ów zostaną po kolei wykonane.

To sprawia, że wykonamy count / 8 (dzielenie całkowite) obrotów pętli, oraz pierwsza iteracja będzie składała się z przekopiowania tylko count % 8 elementów, a więc mamy kopiowanie ⌊count8⌋∗8+count % 8⌊count8⌋∗8+count % 8 elementów, czyli dokłądnie count elementów.

n = (count + 7) / 8 daje nam ⌈count8⌉⌈count8⌉, ale w warunku pętli jest --n, więc już przy pierwszym sprawdzeniu warunku n==⌊count8⌋n==⌊count8⌋ i tyle obrotów pętli się wykona.

Wystarczy więc zrobić tablicę z etykietami i ułożyć je w kolejności takiej jak w pierwotnej funkcji secret i instrukcję goto przenoszącą nas w odpowienie miejsce, by wykonać count % 8 kopiowań, a następnie już goto odpowiadające za skoki w pętli lub wyjście z niej, aby przekopiować odpowiednią ilość elementów.



















void secret_goto(uint8_t *to, uint8_t *from, size_t count){
   static void* _case[] = {&&case_0, &&case_1, &&case_2, &&case_3, &&case_4, &&case_5, &&case_6, &&case_7};
   static void* _loop[] = {&&exit, &&case_0};
   
   size_t n = (count + 7) / 8;
   goto *_case[count % 8];

   case_0: *to++ = *from++;
   case_7: *to++ = *from++;
   case_6: *to++ = *from++;
   case_5: *to++ = *from++;
   case_4: *to++ = *from++;
   case_3: *to++ = *from++;
   case_2: *to++ = *from++;
   case_1: *to++ = *from++;
   goto *_loop[--n > 0];
   exit: return;
}
