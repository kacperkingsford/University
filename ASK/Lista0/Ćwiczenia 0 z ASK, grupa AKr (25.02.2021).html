• x⋅2yx⋅2y ,
• ⌊x2y⌋⌊x2y⌋,
• x mod 2yx mod 2y ,
• ⌈x2y⌉⌈x2y⌉.




























#include <stdio.h>

int a(int x, int y) {
    return x << y;
}

int b(int x, int y) {
    return x >> y;
}

int c(int x, int y) {
    return x & (a(1, y) - 1);
}

//^[n/m] = .[n-1/m] + 1
int d(int x, int y) {
    //problem dla 0
    return b(x - 1, y) + 1;
    
}

    printf("%d \n", a(10, 4));
    printf("%d \n", b(161, 4));
    printf("%d \n", c(10, 2));
    printf("%d \n", d(13, 2));
}

Zadanie 6

Autor: Maria Wyrzykowska

Napisz fragment kodu, który skonwertuje zmienną xx z formatu little-endian do formatu big-endian. Należy użyć jak najmniejszej liczby operacji bitowych.

Big endian - forma zapisu danych, w której najbardziej znaczący bajt umieszczony jest jako pierwszy. Jest ona analogiczna do używanego na co dzień przez ludzi sposobu zapisu liczb.

Little endian - forma zapisu danych, w której najmniej znaczący bajt umieszczony jest jako pierwszy.

Little endian -> big endian:

b0 = (num & 0x000000ff) << 24;
b1 = (num & 0x0000ff00) << 8;
b2 = (num & 0x00ff0000) >> 8;
b3 = (num & 0xff000000) >> 24;

res = b0 | b1 | b2 | b3;

Alternatywne rozwiązanie, o “logarytmicznej złożoności”:
Mając dane bajty ABCD, bęziemy kolejno:

  • zamienimy bajty AB z bajtami CD, otrzymując CDAB
x = (x<<16) | (x>>16)
  • zamienimy bajt C z bajtem D, a jednocześnie bajt A z bajtem B
x = ((0xFF00FF00&x)>>8) | ((0x00FF00FF&x)<<8)

Zadanie 7

Autor: Paweł Zmarzły

Jaką rolę pełnią kody sterujące standardu ASCII o numerach 0, 4, 7, 10 i 12?

0 NULL - no-op lub separator
NULL jest poprawnym znakiem UTF-8, ale nie wystąpi nigdzie w multibyte
4 EOT - koniec transmisji (Ctrl+D)
sterownik terminala zamyka wtedy wejście programu
7 BELL - zamiast wypisywania, powiadom użytkownika (dźwiękiem lub wizualnie)
10 LINE FEED - przesuń kartkę w dół
12 FORM FEED - następna kartka
13 CARRIAGE RETURN - wróć na początek linii

nowa linia
Windows: CR LF
Unix: LF

Ctrl+Shift+U - wprowadzanie w terminalu
